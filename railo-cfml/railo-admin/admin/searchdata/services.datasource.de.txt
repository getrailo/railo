Zeitzone Gleiche wie in Railo Instanz Wenn Sie ein Datum als String definieren, also nicht mit cfqueryparam, enthält dies normalerweise keine Zeitzoneninformation und basiert auf der lokalen Zeit. Die Datenbak interpretiert dann dieses Datum basierent auf seiner eigenen Zeitzone. Wenn sich nun die Zeitzone der Datenbank und von Railo unterscheidet, kann dies zu Problemen führen. Railo erlaubt es hier, dass Sie für eine Datasource eine spezifische Zeitzone definieren können, um diese Problem zu umgehen. Definieren Sie nur eine spezifische Zeitzone wenn Sie sicher sind, dass diese Zeitzone in der Database auch verwendet wird! Dieses setting wird durch das Attribute "timezone" des Tag query überschrieben.
